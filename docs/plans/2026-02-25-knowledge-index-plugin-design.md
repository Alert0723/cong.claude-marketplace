# 知识索引插件设计文档

## 1. 概述

为 Claude Code Marketplace 开发一个选择性知识索引插件，实现跨会话知识管理和按需检索功能。

**核心目标：**
- 低 token 消耗
- 按需检索，不自动加载全部知识
- 用户可控的知识分类（用户级/项目级）
- 首次安装配置提醒

## 3. 核心组件

### 3.1 变更检测器
- 监听文件变化（基于 git diff 或文件系统监听）
- 提取变更内容（新增/修改/删除）
- 过滤忽略文件（node_modules, .git, 等）

### 3.2 重要度分析器
- 分析变更的重要程度（0.0 - 1.0）
- 考虑因素：文件大小、变更行数、文件类型、代码复杂度
- 输出评分 + 简要说明
- **注意：重要度仅用于排序和展示，不作为过滤阈值**

### 3.3 索引生成器
- 生成知识摘要
- 判断归属（用户级/项目级）
- 存储结构化数据

### 3.4 相似度检测器
- 基于文件路径相似度
- 基于内容相似度
- 基于知识摘要相似度
- 检测可合并的知识并提示用户

### 3.5 待定队列管理器
- 管理 pending 状态的知识
- 提供查询/筛选功能
- 状态流转操作

### 3.6 知识检索器
- 在对话启动时根据上下文查询相关知识
- 支持语义搜索和关键词匹配
- 返回最相关的知识片段
- **按需加载，不自动注入全部知识**

## 4. 知识状态流转

| 状态 | 说明 | 用户操作 |
|------|------|----------|
| `pending` | 初始检测到，等待分类 | 查看详情，选择分类 |
| `skipped` | 用户选择跳过 | 可重新归类 |
| `classified` | 已确定用户级/项目级 | 确认学习 |
| `learned` | 已存入知识库 | 可删除 |

## 5. 按需检索流程

```
用户提出问题/请求
    │
    ▼
分析查询意图
    │
    ▼
知识检索器查询
    │
    ├── 用户级知识库 ──┐
    │                   │
    └── 项目级知识库 ──┼─▶ 合并相关结果
                        │
                        ▼
                    注入相关知识到上下文
                        │
                        ▼
                    Claude基于上下文回答
```

## 6. 用户命令

| 命令 | 功能 |
|------|------|
| `/knowledge` | 查看待学习知识，进行分类 |
| `/knowledge:setup` | 首次安装配置 |
| `/knowledge:status` | 查看知识库统计 |
| `/knowledge:search <query>` | 手动搜索知识 |
| `/knowledge:pending` | 查看待定队列 |

## 9. 配置文件

### 首次安装配置
插件首次安装时，应调用 `/knowledge:setup` 提醒用户配置。

### 配置位置
`.claude/cong.claude-marketplace.local.md`

### 配置模板
```json
{
  "autoStart": false,
  "confirmBeforeLearn": true,
  "userKnowledgeDir": "~/.claude/knowledge",
  "projectKnowledgeDir": ".claude/knowledge"
}
```

## 11. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 知识库文件损坏 | 跳过损坏条目，记录日志 |
| 检索无结果 | 返回空结果，不影响对话 |
| 索引写入失败 | 保留在 pending 队列，重试 |
| API 调用超时 | 降级使用本地 grep 搜索 |
